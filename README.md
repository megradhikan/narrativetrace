# NarrativeTrace

Live claim-clustering and coordination-signal detection over the Bluesky AT Protocol firehose.

> **NarrativeTrace surfaces signals for human review — it does not determine truth, label content as disinformation, or identify bots.**

---

## v0.1 — Firehose ingestion

This version connects to the Bluesky public firehose, filters to text posts, and persists them to Postgres.

---

## Prerequisites

- Python 3.11+
- PostgreSQL 14+ (with the `psycopg2` driver)

---

## Local setup

### 1. Create and activate a virtual environment

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

### 2. Set up Postgres

Create the database (the ingestion service creates the table automatically):

```bash
createdb narrativetrace
```

If your Postgres uses a non-default user/password/host, set the `DATABASE_URL` environment variable:

```bash
export DATABASE_URL="postgresql://user:password@localhost:5432/narrativetrace"
```

Copy `.env.example` to `.env` and edit as needed (or export variables manually — the code reads from the environment directly).

### 3. Run the ingestion service

```bash
python -m ingestion.run
```

The service will:
- Connect to the AT Protocol firehose at `bsky.network`
- Filter to text post-create events only (skips likes, follows, reposts, profile updates)
- Persist each post to the `posts` table in Postgres
- Log post count and rate (posts/min) to stdout once per minute
- Reconnect automatically with exponential backoff (1s → 60s) if the connection drops

Press `Ctrl-C` to stop gracefully.

### 4. Verify posts are flowing

```sql
psql narrativetrace
SELECT count(*), min(created_at), max(created_at) FROM posts;
```

## Running tests

Tests run against a pre-recorded fixture file — no live firehose connection required.

**First-time only:** capture the fixture (requires a live connection for ~30 seconds):

```bash
python -m ingestion.capture_fixture
```

This writes `fixtures/firehose_sample.json` (~500 posts).

Then run the test suite at any time, including in CI without network access:

```bash
pytest
```

## Configuration

| Variable | Default | Description |
|---|---|---|
| `DATABASE_URL` | `postgresql://localhost:5432/narrativetrace` | Postgres connection string |
| `FIREHOSE_HOST` | `bsky.network` | AT Protocol firehose host |
| `RECONNECT_BASE_DELAY` | `1` | Initial reconnect wait (seconds) |
| `RECONNECT_MAX_DELAY` | `60` | Maximum reconnect wait (seconds) |
